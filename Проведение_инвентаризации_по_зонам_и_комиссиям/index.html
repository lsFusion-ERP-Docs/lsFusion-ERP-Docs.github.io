Проведение инвентаризации по зонам и комиссиям

Алгоритм проведения инвентаризации по зонам и комиссиям, позволяет анализировать данные по зоне, если её считали несколько комиссий.

По зоне сравниваются данные каждой комиссии, при расхождении данных - пользователь выбирает какое значение вносить в опись.

Заполняем шапку документа инвентаризации, выбираем операцию, склад, проставляем дату.

Параметры заполняются автоматически из настроек операции.

Сохраняем документ.

На вкладке "Описи" в ячейке " Кол-во комиссий для принятия решений" ставим цифру равную количеству комиссий которые проверяют одну и ту же зону.
Нажимаем кнопку [ Выбрать документы ТСД ].

Нас интересует Тип документа "Инвентаризация" и нужный склад. Отмечаем птичкой "Вкл." документы для приема из ТСД и нажимаем [ ОК ].

Примечание

Обратите внимание

Чтобы провести инвентаризацию по зонам и комиссиям, необходимо, чтобы для ТСД были настроены справочники Зон и Комиссий, а также документ инвентаризации, в котором обязательно указание зоны и комиссии. Подробнее о настройках для ТСД читайте в этой статье. Зоны добавятся в Опись, если "Код" зоны и "Код в справочнике 1" в документе ТСД одинаковы. Чтобы этого достичь необходимо одновременное выполнение двух моментов:1) при настройке вида справочника ТСД указать признак "Не выгружать префикс"2) коды вида справочника использовать в кодах Зон и Комиссий

В результате получаем список зон с расхождениями. Справа от зоны видны документы из ТСД, разбитые по комиссиям.

Одна комиссия на зону

Видим, что в колонке "Количество расхождений" не заполнено значение и ячейки подсвечиваются зеленым цветом. В этом алгоритме инвентаризации "расхождение" - это расхождение в кол-во указанном несколькими комиссиями по одной и той же зоне , т.е. расхождения между данными указанными несколькими комиссиями, а если комиссия одна, то и расхождений нет. Переходим справа на вкладку "SKU",так как нет расхождений данные единственной комиссии сразу отображаются в колонке "В опись"

Несколько комиссий проверяет зону

В колонке "Количество расхождений" может быть цифра - сколько расхождений у комиссий и подсвечиваться красным при наличии расхождений.

По этим расхождениям пользователю нужно принять решение, какое из значений вносить в опись.
Переходим на вкладку "SKU", если по зоне есть расхождения, ищем в колонке "В опись" подсвеченные красным цветом ячейки ( или отбираем фильтром "Разногласия (F10)") и проставляем фактическое количество товара либо одно из значений комиссии - после этого у ячейки "В опись" цвет изменится на зеленый, а цифра в колонке "Количество расхождений" уменьшиться.

Так нужно обработать позиции с расхождениями , чтобы в колонке "Количество расхождений" отсутствовало значение и ячейка в колонке стала светиться зеленым.

Чтобы распечатать расхождения по зоне, на которой стоит курсор - пользуемся кнопками "Расхождения(зона)" или "Расхождения(зона) XLSX".

Если хотим видеть все расхождения по инвентаризации по всем зонам, то используем "Расхождения(все)"

Переходим на вкладку "Описи", нажимаем кнопку "Создать описи по зонам". После этого на вкладке "Описи" виден конечный результат по описям, который будет принят к инвентаризации.

В случае ошибки можно исправить количества по описи кнопкой [ Редактировать ].

Далее работаем с документом инвентаризации на вкладке "Сличительная ведомость" ( см. Инвентаризация, ...Переходим в закладку "Сличительная ведомость" )
