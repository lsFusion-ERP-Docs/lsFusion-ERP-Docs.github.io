Подарочные сертификаты

Использование подарочных сертификатов возможно в рамках базового и расширенного модулей. Базовый модуль дает возможность вести учет подарочных сертификатов и получать информацию о суммах проданных по кассе и использованных для оплаты товаров сертификатов в разрезе магазинов. При использовании варианта с расширенной функциональностью подарочный сертификат является товаром, обеспечивается возможность списания сертификатов, их безналичной продажи и формирования отчетности по сертификатам.

Функционал подарочных сертификатов может быть доступен при использовании встроенного POS (модуль "Управление продажами") или кассовых систем УКМ 4 и Set Retail 10.

Базовый модуль

В Администрирование - Настройки на вкладке Подарочные сертификаты доступны опции:

Разрешить погашать сертификат частично - необходимо отметить, чтобы можно было использовать не весь номинал сертификата сразу, а по частям на несколько покупок.

Дата продажи сертификатов, с которой их можно использовать несколько раз - можно указать дату, проданные после которой сертификаты можно будет погашать частично. Сертификаты, проданные до указанной даты, можно будет погашать только одномоментно.

В группах касс необходимо настроить Отдел продажи сертификата и Тип оплаты сертификатом для корректного обмена данными с фискальным регистратором. Для касс VM и Sento данная настройка не нужна.

Создают и просматривают информацию о сертификатах на форме Розничная торговля - Подарочные сертификаты. В блоке Подарочный сертификат отображаются подарочные сертификаты и информация о них - номер, серия, код, дата создания, данные о продаже и погашении, сроке действия и т.д. В блоке Движение подарочных сертификатов отображаются продажи и погашения выбранного сертификата, документы, по которым реализовывался или погашался сертификат можно открыть для просмотра.

Для создания новых сертификатов необходимо нажать кнопку Сгенерировать ПС и заполнить данные в диалоговом окне. Номер сертификата будет генерирваться в соответствии с заданным нумератором. Необходимо указать:

Количество подарочных сертификатов​

Цена подарочных сертификатов - она же является номиналом сертификата

Дата - дата создания сертификата

Срок действия - срок, который указывается в днях и отсчитывается с даты продажи, в течение которого можно погасить сертификат.

В Розничная торговля - Отчет по магазинам доступны колонки с данными продаж по сертификатам.

Для данной конфигурации необходимо подключение следующих модулей:

  • GiftCard Основной справочник ПС
  • ZReportReportGiftCard в отчет по магазинам добавляются поля связанные с ПС
  • POSGiftCard
Расширенная функциональность

Настройка​

В модуле с расширенной функциональностью каждый номинал подарочного сертификата является товаром с типом SKU Номинал подарочного сертификата.

В Администрирование - Настройки на вкладке Подарочные сертификаты вводятся первоначальные настройки:

Разрешить погашать сертификат частично - необходимо отметить, чтобы можно было использовать не весь номинал сертификата сразу, а по частям на несколько покупок.

Дата продажи сертификатов, с которой их можно использовать несколько раз - можно указать дату, проданные после которой сертификаты можно будет погашать частично. Сертификаты, проданные до указанной даты, можно будет погашать только одномоментно.

Группа товаров для сертификатов - задается товарная группа, в которую будут сохранятся все созданные номиналы.

Префикс генерации подарочных сертификатов - при генерации номера сертификата будет использован указанный префикс.

Длина номера при генерации подарочных сертификатов - количество символов в номере сертификата, не включая префикс.

Группа организаций для покупателей сертификатов - здесь можно указать группу организаций, которым можно будет выписывать сертификаты по безналичному расчету.

Основная работа с подарочными сертификатами осуществляется на форме Рабочий стол - Подарочные сертификаты. На вкладке Сертификаты отображаются все сертификаты, которые есть в системе и информация об их выпуске, продаже и использовании.

Создание Подарочных Сертификатов

Чтобы сгенерировать подарочные сертификаты, необходимо сначала создать номиналы сертификатов. Все созданные номиналы можно просмотреть на вкладке Номиналы. Новый номинал создается кнопкой Добавить.

Для нового номинала подарочного сертификата необходимо указать Название, Цену и Срок действия. Будет создан товар с типом SKU Номинал подарочного сертификата, который будет отображаться как на вкладке Номинал формы Рабочий стол - Подарочные сертификаты, так и в справочнике товаров.

Далее идем на вкладку Поступление сертификатов и нажимаем кнопку Добавить чтобы создать документ поступления сертификатов.

В шапке заполняем Номер, Серию, выбираем магазин (Склад) и нажимаем кнопку Добавить. В спецификации появится пустая строка, в которой необходимо указать номинал сертификата. Для этого в строке нажимаем на поле Код (или Номинал) и выбираем номинал Подарочного Сертификата из доступных. Если нет нужного номинала, необходимо нажать кнопку Добавить и ввести все нужные параметры для нового номинала сертификата.

После этих действий необходимо ввести Номер с/по для новых сертификатов, нажать кнопку Сгенерировать ПС и в появившемся диалоговом окне выбрать ответ Да.

Будут созданы сертификаты, их номера будут сгенерированы в соответствии с настройками. Чтобы завершить процесс создания сертификатов необходимо Провести документ.

Сертификаты можно перемещать в другие магазины/склады, однако это не будет отражено как движение сертификата.

Для перемещения ПС необходимо добавить новый документ в «Поступление сертификатов»

Выбрать склад, на который мы будем перемещать, нажать кнопку Переместить ПС, выбрать ПС, которые мы будем перемещать на этот склад, нажать ОК и после этого Провести документ.

Продажа подарочных сертификатов по безналичному расчету​

Для продажи ПС по безналичному расчету необходимо:

  1. Создать розничный прайс для номиналов ПС
  2. В настройках операции продажи ПС (Продажи - Справочники/Операции)на вкладке Отображение свойств отметить Подарочные сертификаты.

На форме Рабочий стол - Выписка подарочных сертификатов нажать кнопку Продажа сертификатов, в открывшемся окне заполнить данные Покупателя, Склада, указать Количество сертификатов для продажи нужных номиналов, нажать ОК.

Автоматически будет создана накладная продажи. В накладной на вкладке Производные документы свойство Создать поставку должно быть неактивным.

Печать счет-фактуры производится из формы Выписка подарочных сертификатов вкладка Печатные формы.

Чтобы выдать сертификаты покупателю, необходимо нажать кнопку Отпуск сертификатов покупателю, отметить номера ПС в открывшейся форме и нажать ОК.

Статус выписки накладной изменится на Выписана, на форме Выписка подарочных сертификатов в блоке Подарочные сертификат отображаются номера проданных сертификатов. На форме Подарочные сертификаты вкладка Сертификаты будет отображена информация о дате и месте продажи сертификатов. При использовании модуля POS lsFusion данные сертификаты уже могут быть использованы для оплаты. При работе c другими системами, могут понадобится дополнительные действия для отправки информации о сертификатах на кассовые серверы.

Списание подарочных сертификатов

Списание испорченных производится на форме Розничная торговля - Списание испорченных ПС, где отображаются документы списания и номера списанных сертификатов. Чтобы списать ПС необходимо создать новый документ - кнопка Добавить.

В документе надо заполнить шапку, кнопкой Добавить создать новую строку, кликнуть в ней и выбрать сертификат, который надо списать. Добавив все испорченные сертификаты, нажмите Сохранить.

Отчеты по подарочным сертификатам​

Для получения отчета для ПС необходимо, зайти Розничная торговля – Отчет по ПС, выбрать период, склад и вариант отчета (.pdf или .xlsx).

В сгенерированных отчетах представлена информация о поступлении, наличии и списаниях испорченных сертификатов на выбранном складе.

список модулей lsf (для специалистов поддержки)

Для данной конфигурации необходимо дополнительное подключение следующих модулей:

  • GiftCardItem модуль привязки ПС к товару
  • GiftCardPurchase Поступление ПС на магазины
  • GiftCardDefect Списание ПС
  • GiftCardSaleInvoice Продажа ПС по безналу
  • GiftCardReport Добавляются форма listGiftCards 'Отчет по ПС' и отчет useGiftCardPrint
