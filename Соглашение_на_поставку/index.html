Соглашение на поставку

Соглашение на поставку - это объект, который описывает условия, согласованные между поставщиком и покупателем, при поставке товара. При помощи соглашений в системе регламентируются цены, по которым будет поставляться товар, допустимые отклонения и прочие параметры, существенные при получении товара на склад.

Для каждого соглашения должно быть задано, какие организации могут использоваться в качестве поставщиков и покупателей. Одно соглашение может действовать как для двух отдельных организаций, так и между различными группами организаций.

У каждого соглашения задается период его действия. В документах, у которых есть дата и время, может быть выбрано только соглашение, у которого дата входит в его период действия. В случае, если условия соглашения изменяются, то чаще всего необходимо добавить новое соглашение, действующее с определенной даты, при этом ограничив этой датой срок действия старого соглашения. В любой момент времени могут одновременно действовать сразу несколько соглашений. В этом случае пользователю в момент ввода соответствующего документа будет предложено выбрать одно из них.

Соглашения используются в заказах и накладных.

Форма соглашения.

Атрибуты

В документе соглашения доступны следующие атрибуты:

  • Наименование. Краткое описание соглашения, по которому его можно идентифицировать.
  • Код. Код соглашения. Используется при интеграции с внешними системами.
  • Нумератор. Нумератор, используемый при автоматической генерации номера и серии соглашения.
  • Номер. Внутренний номер соглашения.
  • Серия. Серия внутреннего номера соглашения.
  • Дата. Дата заключения соглашения.
  • Время. Время заключения соглашения.
  • Дата с. Дата, с которого действует соглашение.
  • Время с. Время, с которого действует соглашение.
  • Дата по. Дата, по которое действует соглашение.
  • Время по. Время, по которое действует соглашение.
  • Валюта. Определяет в какой валюте должны задавать цены на поставляемый товар.
  • Вид цены. Вид цены, который будет использоваться для определения цены на поставляемый товар. Может быть уточнено для групп товаров или конкретных товаров.
  • Условия оплаты. Определяет, какие условия оплаты будут использоваться для взаиморасчетов за поставленный товар.
  • Примечания. Дополнительная символьная информация, касающаяся соглашения.
  • % отклонения по количеству вниз. Определяет допустимый процент отклонения вниз по количеству в поставке относительно количества, указанного в заказе. Например, значение 20 будет ограничивать поставленное количество равным 8, если было заказано 10 штук. Если процент не задан, то ограничение отсутствует.
  • % отклонения по количеству вверх. Определяет допустимый процент отклонения вверх по количеству в поставке относительно количества, указанного в заказе. Например, значение 20 будет ограничивать поставленное количество равным 12, если было заказано 10 штук. Если процент не задан, то ограничение отсутствует.
  • % отклонения по цене вниз. Определяет допустимый процент отклонения вниз по цене в накладной относительно цены, указанной в заказе, или цены для вида, указанного в соглашении, если накладная идет без заказа. Например, значение 20 будет ограничивать цену в поставке равным 8000, если цена была 10000. Если процент не задан, то ограничение отсутствует.
  • % отклонения по цене вверх. Определяет допустимый процент отклонения вверх по цене в накладной относительно цены, указанной в заказе, или цены для вида, указанного в соглашении, если накладная идет без заказа. Например, значение 20 будет ограничивать цену в поставке равным 12000, если цена была 10000. Если процент не задан, то ограничение отсутствует.
  • Запретить оприходование накладных без заказа. Если данная опция включена, то будет запрещено вводить приходные накладные без указания заказа, на основании которого была выписана накладная.
  • Минимальная сумма заказа. Определяет минимальную сумму, по которой может быть сделан заказ в рамках соглашения.
